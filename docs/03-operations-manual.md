# 操作マニュアル｜100億宣言支援AI 管理コンソール

> 事務局スタッフ向け業務ガイド
> 配置先：`~/ynmo-admin/docs/03-operations-manual.md`

---

## はじめに

このコンソールは、**100億宣言企業（約3,135社）の事務局運営を効率化**するための管理画面です。

### このマニュアルの対象
- 100億宣言事務局のスタッフ（5〜10名想定）
- 普段は Excel / Outlook を使っているレベルの方
- プログラミング経験は不要

### このコンソールでできること
- 取り組み事例（PDF）のアップロード・配信スケジュール管理
- マッチング申請の状況確認・相談会の企画
- LINE ユーザーへの個別連絡（チャット式）
- 認可企業 3,135 社の検索・閲覧
- 相談会の運営管理

## 2. 朝のルーティン（毎日の確認手順）

朝出社したら、以下の流れで状況を把握します。**3〜5分**で完了します。

### Step 1：ダッシュボードを開く
サイドバーの「ダッシュボード」をクリック。

### Step 2：KPI カードで全体傾向を把握

画面上部に4枚のカード：

| KPIカード | 見るポイント |
|---|---|
| **登録ユーザー数** | LINE ユーザーの増減（前月比で +N 名と表示） |
| **配信開封率** | 取り組み事例の見られ方（前月比で増減） |
| **今月のマッチング** | 申込件数の推移 |
| **相談会調整開始**（紺色強調） | **今すぐ動くべき件数** ⚠ |

→ 4枚目の**紺色のカード**に数字が入っていたら、**今日の業務が発生**しているサインです。

### Step 3：今後の配信予定を確認

カレンダーに**配信予定日のマーカー（青）**が表示されています。

- カレンダーをクリック → その日の配信内容を確認
- 「>」で次月に移動可能

リスト形式でも「直近3件」が表示されます。

### Step 4：相談会調整開始企業をチェック

右側の「相談会調整開始企業一覧」に**4件以上の申込が集まった企業**が表示されます。

- 経過日数が**赤＋⚠アイコン**（30日超）の企業：**最優先で対応**
- 経過日数が**黄色**（8〜30日）：そろそろ動きたい
- 経過日数が**グレー**（7日以下）：余裕あり

→ 「**相談会作成**」ボタンを押すと、新規相談会作成画面に直行します。

### Step 5：直近の相談会を確認

下部のテーブルで、今後開催予定の相談会を確認。

- 「3日後」「明日」のような相対表記
- 行をクリックで詳細画面へ

## 3. 業務フロー別の操作手順

### A. 取り組み事例の配信（一連の流れ）

#### A-1. 新規事例の登録

1. サイドバー「**取り組み事例**」→ 右上「**+ 新規追加**」
2. 左側の「PDFをアップロード」エリアにドラッグ＆ドロップ（または「ファイルを選択」）
3. 右側のフォームで設定：
   - **配信予定日**：DatePicker で日付・時刻を選ぶ
   - **配信先業界**：18業界からチェックボックスで選択（複数可）
   - **配信先企業規模**：7段階の売上規模から選択（複数可）
4. 「**配信対象：約 N 社**」が下部に表示される（リアルタイム計算）
5. 「**公開予定として保存**」ボタンを押下

#### A-2. 配信予定の確認・変更

1. 「取り組み事例」一覧で「**公開予定**」のステータスをクリックして該当事例を開く
2. 上部に**青いバナー**：「この事例は公開予定です（{配信日}）」
3. 「**公開予定を変更**」ボタンを押下 → **黄色の警告バナー**に切り替わり、編集可能になる
4. 配信日や配信先を修正
5. 「**公開予定を更新**」ボタンで保存

#### A-3. 公開済み事例の確認

1. 「公開中」ステータスの事例を開く
2. 配信日・配信先は**読み取り専用**で表示（変更不可）
3. 過去の配信記録として参照可能

#### A-4. 事例のアーカイブ

1. 古い事例を整理したいとき、「**アーカイブ**」ボタン（i アイコンで説明あり）
2. 一覧から非表示になるが、データは残る
3. 後で「アーカイブ解除」で再表示可能

---

### B. 相談会の企画・運営

#### B-1. 相談会の企画判断

1. ダッシュボードの「**相談会調整開始企業一覧**」を確認
2. 4件以上集まった企業の中から、**経過日数の長い**企業を優先
3. 「**相談会作成**」ボタンで新規相談会作成画面へ直行

#### B-2. 相談会の詳細設定

1. 自動で対象企業が選択された状態で開く
2. 入力項目：
   - **タイトル**（例：「コーリョー建販株式会社 相談会」）
   - **開催日時**（DatePicker）
   - **所要時間**（30 / 45 / 60 / 90分）
   - **Zoom URL**（手動入力）
   - **定員**（仮で15名）
3. 「**保存**」で完了

#### B-3. 相談会のリマインド送信

1. 「相談会」一覧 → 該当の相談会をクリック
2. 「**参加者にリマインド一斉送信**」ボタン
3. 個別にリマインドしたい場合は、参加者一覧の各行の「**Bell アイコン**」

#### B-4. 相談会のキャンセル

1. 詳細画面の「**相談会をキャンセル**」ボタン（赤色）
2. 確認モーダルで「キャンセルする」
3. ⚠ **キャンセル後は再開できません**。新しい相談会を作成してください。

---

### C. マッチング申請の確認

#### C-1. 個別申請を時系列で見たいとき

1. サイドバー「**マッチング申請**」
2. タブで「**個別申請（N件）**」を選択
3. 申請日時順に表示
4. ステータスフィルタで絞り込み：
   - **受付**：処理待ち
   - **相談会割当済**：日程決定済み
   - **クローズ**：完了 or 辞退・期限切れ

#### C-2. 企業ごとの集中度を見たいとき

1. タブで「**企業別集約（N社）**」を選択
2. 同じ企業に何件集まっているかを集計表示
3. 「しきい値以上のみ表示」トグルで4件以上のみに絞れる

#### C-3. スコアの意味

各申請に表示される「**スコア（XX%）**」は、申込者と希望先企業のマッチング度合いを AI が算出した数値です。

- 高いほど（80%以上）：相性が良い、有益な情報交換が期待できる
- 低めでも（60%程度）：参考情報、申込のあった事実が重要

---

### D. 個別ユーザーへの連絡

#### D-1. ユーザーを探す

1. サイドバー「**ユーザー**」
2. フィルタで絞り込み：
   - **売上フェーズ**（7段階）
   - **状態**（新規登録 / 確認待ち / 確認済み）
   - **ロール**（admin / manager / member）
   - **検索**（名前・email・カナ）
3. 行クリックで詳細画面へ

#### D-2. ユーザー詳細画面の使い方

3カラム構成：

| エリア | 内容 |
|---|---|
| **左：プロフィール** | 氏名・会社・売上フェーズ・最終ログイン |
| **中央：履歴タブ** | 配信履歴 / マッチング申請履歴 / アクション履歴 |
| **右：チャット** | 過去のやり取り + 新規送信欄 |

#### D-3. チャット式 push 送信

1. 右側の入力欄にメッセージを入力
2. 「**送信**」ボタン押下（または `Cmd/Ctrl + Enter`）
3. メッセージが**即座に履歴に追加**される
4. 「送信しました」の通知

→ **承認フローはありません**。事務局の判断で直接送信できます。

---

### E. 認可企業マスタの参照

#### E-1. 検索・絞り込み

1. サイドバー「**認可企業マスタ**」
2. 上部のサマリーで「業種上位3つ」「47都道府県」など全体感を把握
3. フィルタ：
   - **業種**（18業種、件数併記）
   - **都道府県**（47都道府県）
   - **売上規模**（7段階）
   - **目標達成年**（スライダーで2025〜2056）
   - **検索**（企業名・法人番号で部分一致）

#### E-2. 企業の宣言PDFを見る

1. テーブルの行をクリック
2. **新規タブで宣言PDF**が開く（中小企業庁公開ページ）

## 4. ステータスの意味

### 取り組み事例

| ステータス | 意味 | 操作 |
|---|---|---|
| **下書き** | アップロード済み、配信日未設定 | 自由に編集可能 |
| **公開予定** | 配信日・配信先が設定済み、配信待ち | 「公開予定を変更」ボタンで編集 |
| **公開中** | 配信完了、ユーザーに届いている | 編集不可（記録のみ） |
| **アーカイブ** | 過去の配信を整理 | 「アーカイブ解除」で復元可 |

### マッチング申請

画面上では業務観点で4つのラベルで表示されます：

| 画面表示 | 意味 | 事務局のアクション |
|---|---|---|
| **待機中** | 申込が来たが、まだ4件未満 | 様子見（何もしなくてOK） |
| **⚠ 要対応** | 同じ企業に4件以上集まった | **相談会の企画を開始** |
| **開催準備中** | 相談会の日程が決定済み | 開催に向けてリマインドなど |
| **終了** | 相談会完了 or 企業辞退・期限切れ | 履歴として残る |

### 事務局スタッフのロール

設定画面のスタッフ管理では4つのロールで権限管理：

| ロール | 権限 |
|---|---|
| **管理者（admin）** | 全機能アクセス可、スタッフ管理・ロール変更可 |
| **マネージャー（manager）** | 配信・マッチング・相談会の運営、設定の閲覧 |
| **メンバー（member）** | 配信・マッチング・相談会の運営 |
| **閲覧のみ（viewer）** | データ閲覧のみ、編集・送信不可 |

### ユーザー

| ステータス | 意味 |
|---|---|
| **新規登録** | LINE で友だち追加されたが、プロファイル確認前 |
| **確認待ち** | プロファイル登録中 |
| **確認済み** | プロファイル確定、配信対象 |

### 相談会

| ステータス | 意味 |
|---|---|
| **企画中** | 日程調整中 |
| **開催予定** | 日程確定、これから開催 |
| **開催済** | 完了 |
| **中止** | キャンセル（**再開不可**、新規作成のみ） |

## 5. よくある操作の Tips

### 配信対象企業数のリアルタイム計算
取り組み事例の配信先設定で、業界×売上規模を選ぶと「**約 N 社**」と動的計算されます。**選んだ瞬間に件数が変わる**ので、対象を絞る目安に。

### 検索・フィルタの組み合わせ
複数フィルタは AND 条件で動作。例：「製造業 × 東京都 × 30〜50億」で絞ると、該当する企業のみ表示。

### キーボードショートカット
- **Cmd/Ctrl + Enter**：チャット送信
- **Tab**：フォームの次のフィールドへ
- **Esc**：モーダルを閉じる

### サイドバーの折りたたみ
左上のハンバーガーアイコンでサイドバーを折りたためます。画面を広く使いたい時に。

### 通知ベル
TopBar 右上の鈴アイコン。新着通知をまとめて確認できます（本実装後に有効化）。

## 6. ステータスバッジの色の見方

| 色 | 意味 |
|---|---|
| 🟦 **青（primary）** | 主要、アクティブ、公開中 |
| 🟧 **オレンジ（warning）** | 受付・要対応・確認待ち |
| 🟩 **緑（success）** | 確認済み・開催準備中 |
| 🟥 **赤（destructive）** | エラー・キャンセル・期限切れ |
| ⬜ **グレー** | 完了・アーカイブ・終了 |

→ **オレンジ**を見たら「対応が必要」、**赤**を見たら「重要・止まっている」、**緑**を見たら「順調」と覚えると早いです。

## 7. トラブルシューティング

### 「ボタンが押せない」「画面が真っ白」
1. **ブラウザをリロード**（Cmd/Ctrl + R）
2. それでもダメなら、**ハードリロード**（Cmd/Ctrl + Shift + R）でキャッシュクリア
3. それでもダメなら、ブラウザのコンソール（F12）でエラーを確認、開発担当に連絡

### 「保存しても反映されない」
本プロトタイプは**永続化されません**（リロードで初期状態に戻る）。本実装後は API 経由で DB に保存される予定です。

### 「アバター画像が表示されない」
ネットワークの一時的な問題。少し待つかリロードで解決することがあります。

### 「フィルタが効かない」
1. 「リセット」ボタンで初期化
2. 1つずつフィルタを設定し直し
3. それでも変な動作なら、リロード

## 9. 用語集

| 用語 | 説明 |
|---|---|
| **100億宣言** | 中小企業庁が認定する売上100億円達成を目指す企業の宣言制度 |
| **マッチング** | LINE ユーザーが「この企業の話を聞きたい」と希望する申込 |
| **しきい値** | 同じ企業に申込が一定数（4件）集まると相談会企画の判断時期になる、その基準値 |
| **相談会** | 事務局が企画する、希望先企業へのオンライン相談機会 |
| **AI再生成** | 取り組み事例の AI 生成（本プロトタイプでは UI 化されず、PDF アップロードのみ） |
| **要対応** | 4件以上集まり、事務局が動くべきタイミング |
| **配信** | 取り組み事例（PDF）を対象ユーザーに送ること |
| **アーカイブ** | データを残しつつ一覧から非表示にする機能 |
